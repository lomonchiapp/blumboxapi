BLUMBOX API

Primeros Pasos

La API de Blumbox es una plataforma courier multi-tenant que permite a operadores de logistica gestionar clientes, casilleros, envios, tracking, recepciones y facturacion desde una sola integracion REST.

Base URL https://api.blumbox.do/v1

Principios generales

Todas las solicitudes y respuestas utilizan JSON (Content-Type: application/json). Las fechas se expresan en formato ISO 8601 con zona UTC.

Autenticacion

La API soporta tres metodos de autenticacion segun el tipo de consumidor:

MetodoUso tipicoHeader
API KeyIntegraciones server-to-serverX-Api-Key
JWT OperadorDashboard, operadoresAuthorization: Bearer ...
JWT ClienteApp movil, portal clienteAuthorization: Bearer ...

Rate Limiting

Todas las rutas estan limitadas a 100 solicitudes por minuto por IP / API Key. Al exceder el limite recibiras un 429 Too Many Requests.

Paginacion

Los endpoints de listado aceptan los parametros page (default 1) y limit (default 20, max 100). La respuesta incluye un objeto meta con informacion de paginacion.

Ejemplo de solicitud

bash
curl -X GET https://api.blumbox.do/v1/shipments \
  -H "X-Api-Key: blx_live_abc123..." \
  -H "X-Tenant-Id: uuid-del-tenant" \
  -H "Content-Type: application/json"

Respuesta paginada

json
{
  "data": [ ... ],
  "meta": {
    "page": 1,
    "limit": 20,
    "total": 142,
    "totalPages": 8
  }
}

Autenticacion

API Key (Operadores / Integraciones)

Las API Keys se crean desde el dashboard del operador. Se envian en el header X-Api-Key junto con el X-Tenant-Id. Las keys de produccion comienzan con blx_live_ y las de prueba con blx_test_.

Las API Keys tienen permisos de operador. No las expongas en aplicaciones cliente.
POST /v1/auth/token

Intercambia una API Key por un token JWT de operador con expiracion de 24 horas.

ParametroTipoRequeridoDescripcion
apiKeystringSiAPI Key del operador

JWT Cliente

POST /v1/customers/login

Inicia sesion de un cliente y devuelve un JWT con expiracion de 7 dias.

ParametroTipoRequeridoDescripcion
emailstringSiCorreo electronico del cliente
passwordstringSiContrasena del cliente

API Key directa

bash
curl -X GET https://api.blumbox.do/v1/shipments \
  -H "X-Api-Key: blx_live_abc123..." \
  -H "X-Tenant-Id: uuid-del-tenant"

Obtener JWT de operador

bash
curl -X POST https://api.blumbox.do/v1/auth/token \
  -H "Content-Type: application/json" \

Clientes y Casilleros

Los clientes representan a los usuarios finales del servicio courier. Al registrarse, cada cliente recibe un numero de casillero unico (ej. BLX-00042) que utiliza como direccion de reenvio.

POST /v1/customers/register

Registra un nuevo cliente y genera su casillero automaticamente. Retorna el objeto del cliente creado.

ParametroTipoRequeridoDescripcion
emailstringSiCorreo electronico
passwordstringSiContrasena (min. 8 caracteres)
firstNamestringSiNombre
lastNamestringSiApellido
phonestringSiTelefono con codigo de pais
idTypestringNoCEDULA | PASAPORTE | RNC
idNumberstringNoNumero de documento
POST /v1/customers/login

Inicia sesion del cliente. Retorna un JWT valido por 7 dias.

GET /v1/customers/me

Obtiene el perfil del cliente autenticado. Requiere JWT de cliente.

PATCH /v1/customers/me

Actualiza el perfil del cliente autenticado. Solo se envian los campos a modificar.

GET /v1/customers

Lista todos los clientes del tenant. Solo para operadores. Soporta paginacion y busqueda.

QueryTipoRequeridoDescripcion
pagenumberNoPagina (default: 1)
limitnumberNoResultados por pagina (default: 20)
searchstringNoBuscar por nombre, email o casillero
GET /v1/customers/:id

Obtiene el detalle de un cliente por su ID. Solo operadores.

PATCH /v1/customers/:id

Actualiza datos de un cliente. Solo operadores.

DELETE /v1/customers/:id

Desactiva un cliente (soft delete). El cliente no podra iniciar sesion. Solo operadores.

GET /v1/customers/:id/shipments

Lista los envios asociados a un cliente. Retorna resultados paginados.

Envios (Shipments)

Un envio representa un paquete en transito dentro del sistema. Cada envio tiene un trackingNumber, una fase actual (currentPhase) y puede contener metadata personalizada.

POST /v1/shipments

Crea un nuevo envio en el sistema. Retorna el envio creado con fase inicial CREATED.

ParametroTipoRequeridoDescripcion
trackingNumberstringSiNumero de rastreo del carrier
customerIdstringSiUUID del cliente destino
referencestringNoReferencia interna u orden
metadataobjectNoDatos adicionales (tienda, peso, etc.)
GET /v1/shipments

Lista envios con paginacion. Los operadores ven todos los envios del tenant; los clientes solo los suyos.

QueryTipoRequeridoDescripcion
pagenumberNoPagina (default: 1)
limitnumberNoResultados por pagina (default: 20)
phasestringNoFiltrar por fase actual
customerIdstringNoFiltrar por cliente
GET /v1/shipments/:id

Obtiene el detalle completo de un envio, incluyendo sus eventos de tracking.

GET /v1/trackings/:trackingNumber

Busca un envio por su numero de rastreo. Util para clientes que solo conocen el tracking del carrier.

Eventos de Tracking

Los eventos de tracking registran cada cambio de estado de un envio. Al agregar un evento, la fase del envio se actualiza automaticamente segun la maquina de estados.

Maquina de estados

CREATED→ RECEIVED→ IN_TRANSIT→ IN_CUSTOMS→ CLEARED→ OUT_FOR_DELIVERY→ DELIVERED
EXCEPTION RETURNED ← Pueden ocurrir desde cualquier fase
Los eventos son idempotentes: si se envia un evento duplicado (mismo tipo + shipmentId + timestamp), se retorna el evento existente con "idempotent": true.
POST /v1/shipments/:id/events

Registra un nuevo evento en el envio. Actualiza automaticamente la fase del envio. Retorna el envio actualizado y el evento creado.

ParametroTipoRequeridoDescripcion
typestringSiFase destino: RECEIVED, IN_TRANSIT, etc.
sourcestringSiSCAN | MANUAL | CARRIER_API | WEBHOOK
rawStatusstringNoTexto descriptivo del evento
locationobjectNoUbicacion donde ocurrio el evento
occurredAtstringNoFecha del evento (default: ahora)
GET /v1/shipments/:id/events

Obtiene la timeline completa de eventos del envio, ordenados cronologicamente.

Agregar evento de tracking

bash
curl -X POST https://api.blumbox.do/v1/shipments/s1h2i3p4-.../events \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer eyJhbGci..." \
  -d '{
  "type": "RECEIVED",
  "source": "SCAN",
  "rawStatus": "Paquete recibido en bodega Miami",
  "location": {
    "ciudad": "Miami",
    "pais": "US",
    "bodega": "MIA-WH1"
  }
}'
json — Respuesta 201
{
  "shipment": {
    "id": "s1h2i3p4-m5e6-n7t8-uuid-000000000001",
    "currentPhase": "RECEIVED",
    "trackingNumber": "1Z999AA10123456784",
    "updatedAt": "2026-03-24T15:00:00.000Z"
  },
  "event": {
    "id": "e1v2t3n4-uid0-0000-0000-000000000001",
    "type": "RECEIVED",
    "source": "SCAN",
    "rawStatus": "Paquete recibido en bodega Miami",
    "occurredAt": "2026-03-24T15:00:00.000Z",
    "location": {
      "ciudad": "Miami",
      "pais": "US",
      "bodega": "MIA-WH1"
    }
  },
  "idempotent": false
}

Pre-Alertas

Las pre-alertas permiten a los clientes notificar al courier sobre paquetes que estan esperando. Cuando el paquete llega, el sistema puede vincular automaticamente la pre-alerta con el envio usando el numero de tracking.

POST /v1/customers/me/pre-alerts

Crea una pre-alerta para el cliente autenticado. Retorna la pre-alerta con estado PENDING.

ParametroTipoRequeridoDescripcion
trackingNumberstringSiNumero de rastreo del carrier
carrierstringNoNombre del carrier (UPS, FedEx, USPS, etc.)
storestringNoTienda de origen
descriptionstringNoDescripcion del contenido
estimatedValuenumberNoValor declarado
currencystringNoMoneda (default: USD)
categorystringNoCategoria del producto
GET /v1/pre-alerts

Lista todas las pre-alertas del tenant. Solo para operadores. Soporta paginacion.

GET /v1/pre-alerts/unmatched

Lista las pre-alertas que aun no se han vinculado con ningun envio. Util para operadores al procesar paquetes entrantes.

POST /v1/pre-alerts/:id/link

Vincula manualmente una pre-alerta con un envio existente.

ParametroTipoRequeridoDescripcion
shipmentIdstringSiUUID del envio a vincular

Crear pre-alerta (cliente)

bash
curl -X POST https://api.blumbox.do/v1/customers/me/pre-alerts \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer eyJhbGci..." \
  -d '{
  "trackingNumber": "1Z999AA10123456784",
  "carrier": "UPS",
  "store": "Amazon",
  "description": "Laptop Dell XPS 15",
  "estimatedValue": 1299.99,
  "currency": "USD",
  "category": "Electronica"
}'
json — Respuesta 201
{
  "id": "p1a2l3r4-t5a6-uid0-0000-000000000001",
  "customerId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "trackingNumber": "1Z999AA10123456784",

Recepciones

Las recepciones documentan la llegada fisica de un paquete a una sucursal. Incluyen medidas, peso y cargos asociados. El peso volumetrico se calcula automaticamente usando la formula: (L x W x H) / 5000.

POST /v1/receptions

Registra la recepcion de un paquete en una sucursal. Retorna la recepcion con peso volumetrico calculado.

ParametroTipoRequeridoDescripcion
shipmentIdstringSiUUID del envio
branchIdstringSiUUID de la sucursal
weightLbsnumberSiPeso real en libras
lengthCmnumberNoLargo en centimetros
widthCmnumberNoAncho en centimetros
heightCmnumberNoAlto en centimetros
notesstringNoObservaciones
POST /v1/receptions/:id/charges

Agrega un cargo a la recepcion (flete, seguro, manejo, impuestos, etc.).

ParametroTipoRequeridoDescripcion
conceptstringSiConcepto del cargo
amountnumberSiMonto en USD
typestringSiFREIGHT | HANDLING | INSURANCE | TAX | OTHER
PATCH /v1/receptions/:id/status

Cambia el estado de la recepcion.

ParametroTipoRequeridoDescripcion
statusstringSiPENDING | READY | DELIVERED | RETURNED

GET /v1/branches/public

Calculadora de Tarifas

Calcula el costo de envio basado en peso real, peso volumetrico y las tarifas configuradas por el operador. El sistema utiliza el mayor entre peso real y volumetrico como peso facturable.

POST /v1/calculator/estimate

Calcula una estimacion de tarifa. No requiere autenticacion. Retorna el desglose completo del calculo.

ParametroTipoRequeridoDescripcion
weightLbsnumberSiPeso real en libras
lengthCmnumberNoLargo en cm
widthCmnumberNoAncho en cm
heightCmnumberNoAlto en cm
originZonestringSiZona de origen (ej. MIA)
destZonestringSiZona de destino (ej. SDQ, STI)

Calcular tarifa

bash
curl -X POST https://api.blumbox.do/v1/calculator/estimate \
  -H "Content-Type: application/json" \
  -H "X-Tenant-Id: t1e2n3a4-n5t6-id77-uuid" \
  -d '{
  "weightLbs": 5.5,
  "lengthCm": 40,
  "widthCm": 30,
  "heightCm": 20,
  "originZone": "MIA",
  "destZone": "SDQ"
}'
json — Respuesta 200
{
  "actualWeight": 5.5,
  "volumetricWeight": 4.63,
  "billableWeight": 5.5,
  "pricePerLb": 3.50,
  "flatFee": 2.00,
  "total": 21.25,
  "currency": "USD"
}

Tablas de Tarifas

Los operadores pueden configurar tablas de tarifas personalizadas por zona de origen/destino. Cada tabla contiene tiers (rangos de peso) con sus respectivos precios por libra.

Solo operadores con permisos de administrador pueden gestionar tablas de tarifas.
POST /v1/rate-tables

Crea una nueva tabla de tarifas.

ParametroTipoRequeridoDescripcion
namestringSiNombre de la tabla
originZonestringSiZona de origen
destZonestringSiZona de destino
flatFeenumberNoCargo fijo por envio (default: 0)
GET /v1/rate-tables

Lista todas las tablas de tarifas del tenant.

POST /v1/rate-tables/:id/tiers

Agrega un rango de peso a la tabla de tarifas.

ParametroTipoRequeridoDescripcion
minWeightnumberSiPeso minimo del rango (lbs)
maxWeightnumberSiPeso maximo del rango (lbs)
pricePerLbnumberSiPrecio por libra en USD
PATCH /v1/rate-tables/:id

Actualiza una tabla de tarifas existente.

DELETE /v1/rate-tables/:id

Elimina una tabla de tarifas y todos sus tiers.

Crear tabla de tarifas

bash
curl -X POST https://api.blumbox.do/v1/rate-tables \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer eyJhbGci..." \
  -d '{

Verificar firma (Node.js)

javascript
const crypto = require('crypto');

function verifyWebhook(body, signature, secret) {
  const expected = crypto
    .createHmac('sha256', secret)
    .update(JSON.stringify(body))
    .digest('hex');

  return `sha256=${expected}` === signature;
}

// Uso en Express
app.post('/webhooks/blumbox', (req, res) => {
  const sig = req.headers['x-blumbox-signature'];
  const event = req.headers['x-blumbox-event'];

  if (!verifyWebhook(req.body, sig, WEBHOOK_SECRET)) {
    return res.status(401).send('Firma invalida');
  }

  console.log(`Evento: ${event}`, req.body);
  res.status(200).send('OK');
});

Fases de Tracking

Cada envio progresa a traves de una maquina de estados definida. Las transiciones son unidireccionales en el flujo principal, excepto por las fases de excepcion que pueden ocurrir desde cualquier punto.

Fases del envio

FaseDescripcionTransiciones permitidas
CREATEDEnvio registrado en el sistemaRECEIVED, EXCEPTION
RECEIVEDPaquete recibido en bodega de origenIN_TRANSIT, EXCEPTION, RETURNED
IN_TRANSITPaquete en camino al destinoIN_CUSTOMS, EXCEPTION
IN_CUSTOMSPaquete retenido en aduanasCLEARED, EXCEPTION, RETURNED
CLEAREDPaquete desaduanado exitosamenteOUT_FOR_DELIVERY, EXCEPTION
OUT_FOR_DELIVERYEn ruta de entrega al clienteDELIVERED, EXCEPTION, RETURNED
DELIVEREDEntregado al clienteEstado final
EXCEPTIONProblema o incidenciaCualquier fase anterior, RETURNED
RETURNEDPaquete devuelto al remitenteEstado final

Diagrama de flujo

CREATED→ RECEIVED→ IN_TRANSIT→ IN_CUSTOMS→ CLEARED→ OUT_FOR_DELIVERY→ DELIVERED

Las fases EXCEPTION y RETURNED pueden activarse desde cualquier fase intermedia.

Flujo tipico de un envio

Ejemplo de timeline
Mar 24, 08:00  CREATED
  Envio registrado - Amazon orden #2026-0042

Mar 24, 14:00  RECEIVED
  Paquete recibido en bodega Miami (MIA-WH1)

Mar 25, 06:00  IN_TRANSIT
  Consolidado enviado via aerea MIA -> SDQ

Mar 25, 18:00  IN_CUSTOMS
  En proceso de desaduanaje - DGA

Mar 26, 10:00  CLEARED
  Desaduanado exitosamente

Mar 26, 14:00  OUT_FOR_DELIVERY
  En ruta - Sucursal Naco, Santo Domingo

Mar 26, 16:30  DELIVERED
  Entregado a Juan Perez - Firma recibida

Errores

La API utiliza el formato RFC 7807 (Problem Details) para reportar errores. Todas las respuestas de error incluyen un cuerpo JSON con la estructura estandarizada.

typestringURI que identifica el tipo de error
titlestringNombre legible del error
statusnumberCodigo HTTP del error
detailstringDescripcion especifica del problema
instancestringRuta de la solicitud que genero el error

Codigos de estado HTTP

CodigoSignificadoDescripcion
